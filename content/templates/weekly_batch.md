# 週次バッチ生成プロンプト（Claude Codeに投入）

> 毎週日曜日にClaude Codeに投入するプロンプト

---

## 指示

今週分のTikTokコンテンツを7本生成して。

`content/templates/prompt_template.md` のルールに従え。

---

## カテゴリ配分（曜日別）

```
月曜: あるある（40%枠）
火曜: あるある（40%枠）
水曜: 転職（25%枠）
木曜: あるある（40%枠）
金曜: 給与（20%枠）
土曜: トレンド（10%枠）
日曜: 転職（25%枠）
```

---

## CTA配分（8:2ルール）

```
月曜: ソフトCTA（「保存してね」）
火曜: ソフトCTA（「フォローで続き見れる」）
水曜: ソフトCTA（「共感したらいいね」）
木曜: ソフトCTA（「この情報役立つよね」）
金曜: ソフトCTA（「保存推奨」）
土曜: ソフトCTA（「シェアして」）
日曜: ハードCTA（「LINEで相談できるよ。プロフから登録」）
```

**重要:** 10投稿中8投稿はソフトCTA。ハードCTAは週1回（日曜のみ）。

---

## 生成手順

### Step 1: 台本生成
各曜日ごとに台本をJSON形式で生成。

出力先: `content/generated/{日付}_{ID}.json`

例: `content/generated/20260223_A05.json`

### Step 2: スライド画像生成
生成した各台本に対して、以下のコマンドを実行:

```bash
python3 scripts/generate_slides.py --json content/generated/20260223_A05.json
```

7本すべての台本に対して実行。

### Step 3: Slack通知
すべての生成が完了したら、以下のコマンドを実行:

```bash
python3 scripts/notify_slack.py --message "✅ 今週分の投稿コンテンツ準備完了（7本）
月: A05 - {フック}
火: A06 - {フック}
水: B03 - {フック}
木: A07 - {フック}
金: C03 - {フック}
土: T01 - {フック}
日: B04 - {フック}

全42枚のスライド画像生成済み。
PROGRESS.mdに記録済み。"
```

---

## 前回のパフォーマンスを参考にする

`PROGRESS.md` を確認し、前回の投稿パフォーマンスデータがあれば:
- 再生数1万超え → ★マーク。類似パターンを優先。
- 再生数500未満 → ✕マーク。同じパターンを避ける。

**例:**
- 「師長にAIで見せたら」が1万再生 → 「先輩にAIで見せたら」「彼氏にAIで見せたら」など類似を増やす
- 「AIに○○を聞いてみた」が500未満 → このパターンを避ける

## 注意事項

- **ペルソナチェック必須**: すべての台本で「ミサキが止まるか？」を確認
- **架空設定のみ**: 実在人物・施設名は絶対に使わない
- **法的制約遵守**: 職業安定法、医療広告ガイドラインを厳守
- **CTA 8:2ルール厳守**: 日曜以外はソフトCTAのみ

---

## エラー時の対応

- 画像生成エラー → ベース画像のパスを確認
- JSON形式エラー → prompt_template.md の出力形式に従っているか確認
- Slack通知エラー → .env の SLACK_WEBHOOK_URL を確認

3回失敗したら代替案を提示せよ。
